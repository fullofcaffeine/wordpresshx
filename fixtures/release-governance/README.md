# Release-governance rehearsal fixtures

SDK-003 uses `scenarios.json` to exercise four policy paths with the currently
named owner: non-sensitive public issue triage, sensitive intake while the
private channel is disabled, an attempted stable publication while mandatory
gates are open, and rejection of an attempted immutable-artifact overwrite in
favor of a new replacement patch.

Every identity is deterministic and non-secret. Candidate package hashes are
the SHA-256 digests of `wordpresshx-sdk-<version>-bad` and
`wordpresshx-sdk-<version>-good` synthetic test strings, `simulationOnly` is
always true, and the WordPress catalog
digest is inventory evidence rather than a support claim. Passing the rehearsal
proves that policy decisions fail closed; it does not exercise registry
credentials, receive a real vulnerability, install a release package, or satisfy
the production release/rollback blocker.

`expected/rehearsal.json` is a content-digested golden report generated by
`scripts/release/test-governance.py`.
